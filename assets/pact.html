A pact between To Do Android and To Do API

Requests from To Do Android to To Do API

  • GET REQUEST given test GET

  • POST REQUEST given test POST

Interactions

Given test GET, upon receiving gET REQUEST from To Do Android, with

{
  "method": "GET",
  "path": "/todos"
}

To Do API will respond with:

{
  "status": 200,
  "headers": {
    "Content-Type": "application/json"
  },
  "body": [
    {
      "id": 1,
      "text": "Pick up pickles",
      "checked": false
    },
    {
      "id": 2,
      "text": "Pick up kids",
      "checked": true
    }
  ]
}

Given test POST, upon receiving pOST REQUEST from To Do Android, with

{
  "method": "POST",
  "path": "/todos",
  "headers": {
    "Content-Type": "application/json"
  },
  "body": {
    "text": "Pick up pickles",
    "checked": false
  }
}

To Do API will respond with:

{
  "status": 201,
  "body": {
    "text": "Pick up pickles",
    "checked": false
  }
}
